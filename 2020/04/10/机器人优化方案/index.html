1.机器人管理服务器

​ 需求:

一.把游戏分为低中高档位 1毛到5毛为低档 5毛(不包含5毛)以上到2元为中档 2元以上(不包含2元)为高档。程序自动识别，在同档位可随机进入任务游戏，不可越档。

二.底档机器人自动携带能量范围为50-150 超过200-300自动下分到初始值

​ 中档:携带200-300 下分450-550

​ 高档:携带500-800 下分1000-2000

三. 可设置部分机器人可主动开桌和玩家或机器人对战

​ 部分机器人只与玩家对战

  • 说明

  1. 机器人要分档1毛 1块 3块(区分疲劳值和能量);

  2. 机器人要支持的新玩法加入方式;

  3. 开放了的空间进入机器人凑桌;

  4. 每个空间的活跃桌数(单独控制);

  5. 机器人自带游戏开始时间和结束时间，游戏时间();(24小时分4个时段 有3个上班时段(7h) 1个休息时段(3h)-后台配置);

  6. 机器人主动发起解散;

  7. 后台开启机器人权限后,管理服务器主动加载对应数量的机器人到该空间俱乐部准入列表,管理服务器开始从数据库加载机器资源，并模拟机器人进行登陆,进空间,进游戏等操作;

  8. 依据分档设置上下分阈值(针对能量)。举例:

    机器人A 底档 能量 0-1块 上限(500) 下限(50) 如果当前机器人的能量低于50，机器人主动充值接口,如果能量高于500则发起提现操作;充值和提现金额可设置为200~400的随机值

  9. 需要区分是否是进入公开房玩,公开房只与玩家玩

  • 枚举
  1. 游戏模式
  1. 机器人状态
  1. 分档

注: 机器人随机一个班次，且一天内只执行一次;

2.数据库设计

  1. 机器人对象表(AndroidLockInfo)

    字段 数据类型 说明
    UserID int 用户ID
    LogonPass nchar(32) 用户密码
    AndroidStatus int 状态【0:空闲1:使用】

  1. 机器人属性表(AndroidBaseInfo)

    字段 数据类型 说明
    UserID int 用户ID
    PointLevel int 分档【0,1,2】
    GameMode int 游戏模式【0,1,2】
    SpaceCode int 空间编码
    WorkStartTime datetime 上班时间
    WorkOverTime datetime 下班时间
    RestStartTime datetime 休息开始时间
    RestStartTime datetime 休息结束时间
    PlayType int 公开房/空间

  1. 机器人组件配置表(AndroidDllServerInfo)

    字段 数据类型 说明
    KindID int 游戏标识
    GameName nvarchar(32) 游戏名称
    AndroidDllName nvarchar(31) 组件DLL名称
    ServerVersion int 服务器版本
    ClientVersion int 客户端版本

  1. 空间配置表(AndroidSpaceConfig)

    字段 数据类型 说明
    SpaceCode int 空间编码
    RobotCount int 机器人数量
    ActiveTableCount int 活跃桌数
    FatigueLowCount int 疲劳值底档机器人数量
    FatigueMiddleCount int 疲劳值中档机器人数量
    FatigueHighCount int 疲劳值高档机器人数量
    EnergyLowCount int 能量底档机器人数量
    EnergyMiddleCount int 能量中档机器人数量
    EnergyHighCount int 能量高档机器人数量
    AllowAndroidBeat int 是否容许机器人对打

    AllowAndroidBeat权限未开,那机器人只与玩家对战
